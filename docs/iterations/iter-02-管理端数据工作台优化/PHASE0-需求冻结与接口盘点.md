# Phase 0：需求冻结与接口盘点

## 1. 阶段结论

Phase 0 已完成。学校数据工作台第一版范围冻结为：

- 新增一个管理端主入口：`学校数据工作台`。
- 围绕“学校 -> 学院 -> 专业方向 -> 年份数据”组织数据。
- 第一版只做统一查看、问题定位、数据可视化和跳转维护。
- 不删除旧 CRUD，不重做审核中心，不做复杂内联编辑。
- 旧 CRUD 暂时保留为“原始数据维护 / 高级维护”兜底。

## 2. 第一版范围

### 必须做

- 学校搜索和筛选。
- 学校级 KPI 总览。
- 学院 / 专业方向矩阵。
- 专业方向近三年数据状态。
- 完整度 A/B/C/D 展示。
- 数据覆盖率图表。
- 选中专业方向后的右侧详情面板。
- 跳转原始 CRUD 维护数据。
- 跳转审核中心处理待审核数据。

### 暂不做

- 暂不做右侧详情里的复杂内联编辑。
- 暂不重构所有 RuoYi CRUD 页面。
- 暂不删除旧菜单。
- 暂不完整重做审核中心。
- 暂不做采集任务调度重构。
- 暂不做跨学校大屏式可视化。

## 3. 菜单决策

当前 postgrad 菜单已有：

- `考研择校`
- `学校管理`
- `学院管理`
- `招生专业管理`
- `复试线数据`
- `招生计划数据`
- `拟录取数据`
- `数据体检`
- `采集任务`
- `数据审核中心`

已有菜单简化脚本将菜单分为：

- `基础数据`
- `招生数据`
- `数据审核`
- `数据运维`

第一版新增菜单建议：

```text
考研择校
├─ 学校数据工作台
├─ 基础数据
├─ 招生数据
├─ 数据审核
└─ 数据运维
```

推荐菜单定义：

```text
menu_name: 学校数据工作台
parent_id: 2000
order_num: 1
path: workspace
component: postgrad/workspace/index
perms: postgrad:workspace:view
icon: dashboard
```

旧 `学校管理` 后移到 `基础数据` 下，不作为日常主入口。

## 5. 接口缺口

现有 `/postgrad/school/{id}/overview` 当前主要返回：

- 学校基础信息。
- 学院统计。
- 专业列表。
- 专业是否有复试线、招生计划、拟录取。

它缺少工作台需要的：

- 全局 KPI。
- 学校列表里的完整度摘要。
- 学校列表里的待审核数量。
- 学校列表里的缺失任务数量。
- 专业方向按年份展开的数据状态。
- 专业方向完整度 A/B/C/D。
- 专业方向待审核数量。
- 专业方向采集任务数量。
- 可视化图表需要的聚合数据。

因此 Phase 2 需要新增工作台聚合接口：

```text
GET /postgrad/workspace/stats
GET /postgrad/workspace/schools
GET /postgrad/school/{id}/workspace
```

## 6. 完整度口径冻结

第一版完整度使用全局统一口径：

1. 如果 `program_year_data_quality` 有当前专业当前年份记录，优先使用记录中的 `completeness_level`。
2. 如果没有质量记录，按 score / plan / result 动态计算。
3. 不允许简单写成：

```sql
coalesce(q.completeness_level, 'C')
```

动态计算口径与现有推荐 SQL / 同步 SQL 保持一致：

```text
A：复试线存在，拟录取最低分/最高分/平均分存在，且招生计划或统考名额或录取人数至少有一个存在
B：复试线存在，且平均分/最低分/招生计划/统考名额至少有一个存在
C：复试线存在，但缺少更多关键录取或计划信息
D：缺复试线或严重缺失
```

工作台 Phase 2 后端实现时应把该逻辑封装为可复用 SQL 片段或服务方法，避免再出现多处定义。

## 7. 默认年份范围

第一版默认展示近三年：

```text
2024、2025、2026
```

原因：

- 足够覆盖当前择校判断。
- 降低工作台接口和页面矩阵复杂度。
- 后续可以扩展为年份选择器。

验收时如果某专业只有某一年数据，不强制补齐其他年份，只展示缺失状态。

## 8. 第一版跳转策略

工作台第一版不直接编辑数据，所有维护动作跳转到现有页面：

| 操作 | 跳转目标 |
| --- | --- |
| 编辑专业基础信息 | 招生专业管理 |
| 维护复试线 | 复试线数据 |
| 维护招生计划 | 招生计划数据 |
| 维护拟录取 | 拟录取数据 |
| 查看完整度 | 数据体检 |
| 创建采集任务 | 采集任务 |
| 处理待审核 | 数据审核中心 |

跳转时尽量携带：

```text
schoolId
collegeId
programId
programCode
year
status=pending
```

如果旧 CRUD 暂时无法消费 query 参数，Phase 1/3 可以先完成跳转，Phase 5 再补自动带筛选。
