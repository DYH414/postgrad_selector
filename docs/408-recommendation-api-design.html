408 考研推荐平台原型接口设计

面向当前 Vue 原型：首页快速推荐、推荐结果、院校详情、对比与备选、收藏历史。目标是把“页面上需要什么数据”落成后端接口契约。

RuoYi AjaxResult 响应 App 端接口前缀 /app 数据来源：N诺（第三方整理） 文档日期：2026-05-23

原型页面与接口边界

全站必须固定返回并展示三条风险提示： 复试线不是最低录取分；推荐学校不代表只有这些学校可以报；N诺数据可能遗漏、过时或错误，最终以院校官方公告为准。

通用约定

认证方式

App 端使用独立用户体系。登录后前端保存 token，请求头携带：

Authorization: Bearer {token}

未登录访问需要用户态数据的接口时返回 code != 200，msg = "未登录"。

响应格式

沿用 RuoYi 的 AjaxResult，前端判断业务成功应看 code，不能只看 HTTP 200。

{
  "code": 200,
  "msg": "操作成功",
  "data": {}
}

POST /app/recommendation/generate

提交快速推荐表单。字段名称直接服务当前原型，重点把“目标分数”改为明确的 estimatedScore（预计初试总分）。

{
  "estimatedScore": 300,
  "examCombo": "22408",
  "targetRegions": ["福建", "广东"],
  "studyMode": "any",
  "majorDirections": ["081200", "085404"],
  "riskPreference": "balanced",
  "includeUnverified": false,
  "includeIncompleteData": true,
  "pageSizePerGroup": 12
}
字段 必填 说明
estimatedScore 是 预计初试总分，408 默认按 500 分制校验，建议范围 150-500。
examCombo 是 11408 或 22408，后端必须基于科目组合过滤，不只看“408”。
targetRegions 否 省份/地区数组，空数组表示不限。
riskPreference 是 决定冲刺/稳妥分组数量和排序权重，不改变数据事实。
includeIncompleteData 否 是否展示数据不足分组。默认 true，但不能混进主推荐分组。

收藏与备选分组扩展

现有收藏接口只有增删查。原型里的“冲刺清单 / 稳妥清单 / 保底候选”需要给收藏加分组、备注、排序。

PATCH /app/favorites/{programId}

{
  "groupType": "balanced_sprint",
  "note": "专业实力强，但分数差距偏大",
  "sortOrder": 20
}
字段 建议落库 说明
group_type user_favorite_program.group_type 收藏分组：冲刺、稳中偏冲、稳妥、保底、自定义。
note 已有字段可复用 用户自己的备注。
sort_order user_favorite_program.sort_order 拖拽排序使用。

关键数据模型

推荐接口不建议直接把后台管理表原样暴露给前端。建议由 Service 聚合 school、college、program、program_subject、admission_score、admission_plan、admission_result、program_year_data_quality，输出面向页面的 DTO。
数据 来源表/字段 接口输出字段
学校与专业基础信息 school、college、program schoolName、province、collegeName、programCode、programName
考试组合 program_subject + subject examCombo、subjectCodes、examSubjectsLabel
复试线 admission_score.score_line scoreLine、scoreLineGap = estimatedScore - scoreLine
拟录取统计 admission_result admissionLow、admissionHigh、avgAdmittedScore、avgScoreGap
招生计划 admission_plan planCount、unifiedExamQuota、retestCount、admittedCount、retestAdmissionRatio
N诺数据完整度 program_year_data_quality dataCompleteness、dataCompletenessText
推荐快照 recommendation_log request、result、ruleVersion、dataVersion
用户收藏 user_favorite_program favoriteId、groupType、note、sortOrder

数据完整度定义

等级 定义 推荐策略
A 含复试线、拟录取区间、人数等字段。 可进入主推荐池。
B 含主要分数字段，部分字段缺失。 可进入主推荐池，但必须展示缺失说明。
C 仅有复试线或基础字段。 默认进入“数据不足”分组，不进入主推荐池。

开发优先级

P0：先让原型跑通

  • GET /app/recommendation/options
  • POST /app/recommendation/generate
  • GET /app/recommendation/result/{id}
  • GET /app/programs/{programId}/detail
  • GET /app/programs/compare

P1：备选清单可用

  • 收藏分组：冲刺/稳妥/保底。
  • 收藏备注与排序。
  • 结果导出为清单。
  • 历史推荐可重新载入。

P2：体验增强

  • 院校/专业搜索建议。
  • 多年度趋势筛选。
  • 官方来源补录后覆盖 N诺字段。
  • 对比列拖拽排序与持久化。

验收清单

检查项 通过标准
考试组合 所有卡片、详情、对比都显示完整组合：11408/22408 + 科目中文。
适配等级 主要表达使用“冲刺、稳中偏冲、稳妥候选、保底候选”，不以星星作为主判断。
分数差距 对比表与详情页都展示与复试线差距、与拟录取均分差距。
风险提示 首页使用须知、推荐结果顶部黄条、详情页风险提示三处都出现固定风险文案。
数据来源 文案统一为“N诺数据完整度”，不写“官方可信度 A”。
业务成功判断 前端统一判断 code === 200，失败时展示 msg。
